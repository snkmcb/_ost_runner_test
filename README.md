# ost-runner-test — OpenStrata hosted source-CI E2E fixture

Public end-to-end fixture for
[open-strata](https://github.com/animu-sphere/open-strata)'s generated
GitHub-hosted source CI (v0.9.0, remote-artifact-transport plan Phase 2).

This repo is a product-level contract, exercised on real GitHub-hosted
runners:

- **`plugins/toy`** — a tiny `usd-fileformat` plugin (scaffolded by
  `ost plugin new usd-fileformat toy --extension toy`), verified up to L5
  (golden roundtrip) by the pyramid.
- **`openstrata.ci.yaml`** — the committed support matrix: `pull_request`
  and `main` source cells on `windows-2022` hosted runners, digest-pinned
  to a public runtime SDK on GHCR
  (`oci://ghcr.io/snkmcb/openstrata-cy2026-windows-x86_64-py313-usd`),
  with a version+checksum-pinned `ost` bootstrap.
- **`.github/workflows/`** — generated by `ost ci generate github`, never
  hand-edited. The hosted lane bootstraps `ost` from the pinned release,
  pulls the runtime with `ost artifact pull` (full verification chain),
  builds the plugin from source, and runs `ost plugin test --up-to 5`.

What the lanes prove:

- **PR lane** — read-only source CI on a hosted runner: bootstrap +
  remote runtime pull + build + pyramid, no publish, no secrets.
- **main lane** — the same on push to `main`.
- **cache-disabled run** — set the `OST_CI_DISABLE_CACHE` repository
  variable (or run on a cold cache): the runtime is re-pulled from GHCR
  and the digests must match the cached path bit-for-bit.

Regenerate workflows after editing the matrix:

```sh
ost ci validate --resolve
ost ci generate github --force
```
